Core Idea

Language as a representational substrate for reasoning

Can natural language serve as an intermediate representation for an RL agent navigating a partially observable environment? This project explores using language as a structured bottleneck — encoding visual POMDP observations into text, then using that text to drive policy learning via PPO.

Architecture

Autoencoder + language + PPO pipeline

I developed an autoencoder-like architecture that reconstructs images from structured text representations, enabling feature extraction for a downstream PPO agent. The pipeline has three stages:

  • Perception: Image-based POMDP environments encoded into structured language descriptions via a vision-language model
  • Representation: Text autoencoder reconstructs the visual scene from language, producing latent features
  • Policy: PPO agent consumes latent text-derived features to select actions in the environment

Through systematic analysis, I identified failure modes arising from limited scene context extraction — linking these shortcomings to language model capacity, autoencoder complexity, and the fundamental constraints of text-based representations for spatial reasoning tasks.

Findings

Where language representations succeed and fail

The experiment revealed a fundamental tension: language is powerful for structured, relational reasoning but lossy for spatial and continuous features critical to navigation. This points toward hybrid representations that combine symbolic language structure with geometric feature embeddings as a productive direction for future work.
